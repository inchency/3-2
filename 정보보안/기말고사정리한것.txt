해시
하나의 문자열을 이를 상징하는 더 짧은 길이의 값이나 키로 변환하는 것
MD2, MD4, MD5 있는데 MD5는 16진수 32자리로 되어있음 즉 16의 32개의 결과값 존재
그러나 다른데이터를 입력해도 해시결과값이 있을 수 있고 이를 충돌이라 하는데
충돌이 자주 일어나는 해시는 좋은 해시가 아님
해시의 역할
해시를 통해 X라는 라벨값을 부여받아 해당 사람의 데이터로 직접 접근이 가능하다.
시간이 균일하고 순차탐색보다 속도가 훨씬 빠르다.
보안에서는 해시를 무결성 확인을 위한 알고리즘으로 사용한다.
무결성이란 오직 허가된 사람들에게만 정보가 개방되고, 그들에 의해서만 수정될 수 있음을 보장한다는 의미
RSA(소수 두개의 곱)
MD2 -> 8비트 컴퓨터에 최적화
MD4, MD5 -> 32비트 컴퓨터에 최적화
MD5는 MD4의 확장판 속도가 더 빠른것은 아니지만 데이터 보안성이 더 높음
SHA-> 미국 NSA에 의해 만들어짐 160비트의 값을 생성하는 해시함수 MD4가 발전한형태
MD5보다 좀 느리지만 더 안전

전자서명
서명이란 서명한 사람의 신분을 집약적으로 증명하는 도구로 전자서명도 이와 비슷
우리나라 전자서명법의 정의
전자서명이란 서명자가 해당 전자문서에 서명하였음을 나타내기 위해 전자문서에 첨부되거나
논리적으로 결합된 전자적 형태의 정보를 말한다.
계약을 할 때 사용하는 인감도장은 동사무소와 같은 공공기관에 등록하여 공증을 받은 것으로, 계약서 등의
날인에 사용.
여기서 전자서명이 인감도장이 되고, 두 전자서명은 공인 인증기관에 등록되고 검증되어 사용.
전자서명 원리
철수가 자기 비밀키로 암호화한다(해시값으로) 이걸 영희에게보내면 영희는 해시값을 철수의 공개키로 복호화하고
원문 문서를 해시한 값과 비교한다. 복호화한 해시값과 전달된 편지에서 구한 해시값이 일치하면
철수로부터 온것이 맞고 위조되지 않았다.

전자서명이 제공하는 기능.
위조 불가 : 서명자만이 서명문을 생성 할 수 있음
인증 : 서명문의 서명자를 확인
재사용 불가 : 서명문의 해시값을 전자서명에 이용하므로 한 번 생성된 서명을 다른 문서의 서명으로 사용 불가
변경 불가 : 서명된 문서는 내용을 변경할 수 없기 때문에 데이터 변조 안됨을보장하는 무결성 만족
부인 방지 : 서명자가 나중에 서명한 사실을 부인 할 수 없음.

연구하는 학문
