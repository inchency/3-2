클라우드 컴퓨팅
PC로 시간 장소에 구애받지 않고 서비스를 제공한다.
사용자가 원하는 OS가 탑재되어 있는 컴퓨터 혹은 서버를 인터넷 상에서 제공한다(compute)
1. 컴퓨트 서비스
2. 스토리지 서비스
편리함에 따라 사용량 증가 -> 보안 문제 증가
위험요소
1. 가상 머신들이 상호연결 (하이퍼바이저 감염시 연결된 모든 가상머신에 감염위험)
2. 물리적 자원에 장애 발생시 (자원을 공유하는 모든 사용자의 서비스 중단 위험)
3. 데이터 저장 및 관리 (정보가 이동, 수정되어도 확인이 불가능, 정보유출 가능성)
4. 다양한 단말의 접속 (각각의 단말이 갖는 보안 위협을 그대로 가짐)
5. 인증 및 접근 제어 (대용량 데이터의 분산 저장)
클라우드 컴퓨팅 보안
1. 플랫폼 - 접근제어, 사용자인증
2. 데이터 및 스토리지 - 검색 가능 암호시스템, PPDM, 기밀성, 데이터 태깅
3. 네트워크 - 암호화 프로토콜, 보안장치
4. 단말 - TPM, Virtualization Security, Renewable Security
하이퍼바이저기반 보안 기술 - 클라우드 환경 공격 탐지/차단기법
1. VM Introspection - 가상머신과 하이퍼바이저의 내부 정보를 분석하는 역할을 수행해 보안 상태를 점검하고 공격
을 탐지하는 시스템
2. Agentless Virtual Security Appliance - 독립된 보안 전용 가상머신을 이용하여 다양한 보안 기능을 서비스 하는 시스템
