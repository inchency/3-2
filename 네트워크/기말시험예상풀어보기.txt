종단점 인증에 대해 서술하시오
종단점 인증이란 하나의 통신 개체가 다른 개체에게 자신의 신분을 컴퓨터 네트워크상으로 증명하는 작업이다.
예를 들자면 전자메일을 보낼 때 사용자가 서버에 신원을 입증하는 것이다.
인증프로토콜 ap1.0
단순히 자기가 누구다 밝히는 것 (제3자가 구라칠수도있음)
인증프로토콜 ap2.0
자신의 IP를 보내 인증하는 방법 하지만 IP스푸핑을 통해 IP를 위조 해서 보낼 수 있음
인증프로토콜 ap3.0
비밀번호를 사용. 하지만 역시 비밀번호를 도청 당하면 위험
인증프로토콜 ap3.1
비밀번호를 암호화해서 사용 하지만 역시 재생공격에 취약 암호화된 비밀번호를 도청해 그대로 사용
인증프로토콜 ap4.0
논스라는 평생 한번 사용되는 숫자를 이용한다.
앨리스는 밥에게 엘리스임을 말한다. 그럼 밥은 넌스 R을 선택해 엘리스에게 보낸다. 엘리스는 이 넌스를 암호화하고 암호화된 넌스를
밥에게 보낸다. 밥은 이 메세지를 복호화 해서 자신이 보낸 넌스와같은지를 확인하여 상대방을 인증할 수 있다.
4.0의 단점은 공유하는 대칭비밀키가 필요하다는 것이다. R을 엘리스가 암호화 할 때 필요하기 때문이다.(?)

SSL에 대해 서술하시오
SSL이란 secure sockey layer.
http 상에서 행해지는 거래에 대한 보안을 제공하기 위해 사용된다.
기밀성, 무결성, 종단점 인증같이 향상된 보안 서비스의 TCP 버전이다.
SSL은 응용계층에 존재하며 보안서비스로 강화된 TCP 서비스를 제공하는 전송 프로토콜이다.
SSL은 핸드셰이크, 키유도, 데이트전송 3단계로 되어있다.
핸드셰이크는 서버와 클라이언트가 TCP 연결을 한다. 클라에서 TCP SYN 서버에서 TCPSYNACK 클라에서 TCPACK
클라에서 hello 메세지를 보내면 서버는 자신의 공개키를 담은 인증서를 보낸다.
클라이언트는 서버의 공개키를 이용해 MS를 생성 한 후 이를 암호화해서 EMS를 서버측에 보낸다.
서버는 EMS를 복호화하여 MS를 알게된다. 서버와 클라이언트만 이제 MS를 알게 되었다.
키유도
원칙적으로 MS는 암호화와 데이터무결성을 위한 대칭키로 사용 할 수 있지만 보다 안전하기 위해 MS를 이용해
4개의 키를 생성한다. 1개는 클라이언트가 서버에게 보내는 암호화키, 1개는 클라이언트가 서버에게보내는 MAC키
1개는 서버가 클라한테 보내는 암호화키, 1개는 서버가 클라한테 보내는 MAC이다.
이렇게 앨리스와 밥은 4개의 키를 모두 갖게 된다. 2개의 암호화키는 데이터를 암호화하는데 쓰이고 2개의 MAC키는
데이터 무결성을 확인하는데에 쓰인다.
데이터전송
서버와 클라이언트 모두 4개의 세션 키를 공유하고 있기 때문에 이제 이들은 TCP연결을 통해 서로 안전하게 데이터를
주고받을 수 있다. 또한 무결성을 확인 하기 위해 MAC계산을 할 때 순서번호를 포함시킨다.
SSL 레코드는 타입,버전,길이,데이터,MAC으로 되어 있다 데이터와 MAC은 암호화되어있다.

IPsec이란?
네트워크 계층의 보안을 제공한다.호스트나 라우터와 같은 네트워크 계층의 두 개체간에 IP데이터그램을 보호한다.
AH와 ESP프로토콜이 있는데 AH는 무결성과 출발지 인증을 제공하지만 기밀성은 제공하지않는다.
ESP은 무결성,출발지인증,기밀성을 제공한다. 그래서 ESP이 훨씬 널리 사용된다.
가상사설망 그림 그림
SA - secureassociations
IPsec 데이터 그램을 전송하기 위해 출발지 개체와 목적지 개체는 네트워크 계층에서 논리적 연결을 설립하는데
이 논리적 연결을 SA라 한다. 단방향 연결이다. 그런데 적은 수의 종단점 일경우에는 (라우터 2개만 존재할 때)
SA정보를 직접 입력할 수 있지만 수백 수천개의 IPsec 라우터와 호스트로 이루어진 경우는 불가능하다. 자동화된
방법이 필요하는데 그래서 나온것이 IKE이다. IKE는 SSL 핸드셰이크와 비슷하다.
WEP
무선네트워크에서도 유선네트워크수준의 보안수준을 제공하는 것을 의미한다.
대칭고유키 기법을 이용해 호스트와 무선액세스포인트(기지국) 사이의 인증과 데이터 암호화를 제공하기위해 설계되었다.
무선 호스트가 액세스포인터에게 인증 요청-> 액세스포인터는 128바이트의 넌스값으로 인증요청에 응답
->무선호스트는 액세스포인트와 공유하는 대칭키를 이용해 넌스를 암호화->액세스포인터는 이 넌스를 복호화
이넌스가 일치하면 인증 성공

802.11i
WEP 암호화 보안성이 약해서 나온 것
1. 보안능력의 발견
2. AP가 경유지 역할을 제공하고 클라이언트 지점과 인증서버는 상호 인증하고 함께 마스터키(MK)를 생성한다.
3. 클라이언트스테이션은 PMK를 구한다. 인증서버는 같은 PMK를 얻고 AP로 보낸다.
4. 클라이언트스테이션와 AP는 암호화와 무결성에 이용하는 TK를 얻기 위해 PMK를 사용한다.

SNMP란?
네트워크 장비를 관리하고 감시하기 위해 TCP/IP계층 중 응용계층에서 이용되는 프로토콜이다.
단순한 형태의 네트워크 프로토콜이라 보면 된다.
SNMP를 구성하는 요소로는 매니저와 에이전트가 있다. 매니저는 하나 에이전트는 여러개이며  중앙 집중방식으로 관리를한다.
메니저가 에이전트에게 관리용 메세지를 요청하면 에이전트는 이를 답하고 데이터베이스를 형성한다.(MIB)
CMIP는 너무 복잡해서 단순한 SNMP가 흔히 쓰인다.
SMI는 MIB를 정의하기 위한 일반적인 구조이다. MIB는 계층적으로 조직된 정보의 집합으로 관리자가 조회하거나 설정할 수 있는 정보데이터베이스이다.
request/response mode와 trap mode가 있다.
첫번째 껀 매니저가 요구를 하면 에이전트는 그 순간에 즉시 응답이 온다. trap mode는 매니저가 요청이 없더라도
어떠한 상황이(조건에 맞는 상황) 발생하면(이벤트가 발생하면) 에이전트는 메니저에게 메세지를 보낸다.

OpenStack
대쉬보드를 통해 compute, storage, network 등 거대한 자원 풀을 제어할 수 있는 클라우드 운영체제이다.
오픈스택의 특징은 이러한 자원들을 모두 웹 인터페이스를 통해 관리할 수 있다는 점이다.
오픈스택은 오픈소스이기 때문에 모든 사람들은 자신이 필요한 만큼 구성요소를 추가 할 수 있다.
오픈스택 운영에 있어 중심이 되는 핵심구성요소 몇개를 말해보겠다
nova : openstack의 기본 컴퓨팅 엔진이며 다양한 가상머신과 인스턴스를 구축하고 관리하는 역할을 한다.
swift : 파일 저장 시스템
cinder : 블록 스토리지 요소, swift보다 좀 더 전통적인 저장 방식과 유사함
neutron : 오픈스택용 네트워크 제어 장치, 효율적 통신을 가능하게해줌
horizen : 대시보드 구성 요소, 사용자들에게 웹 인터페이스를 제공
keystone : 인증서비스를 제공, 키스톤에 등록된 사용자들만 오픈스택을 사용 할 수 있음
glance : iso같은 os설치용 파일을 제공함. glance에 있는 이미지를 이용해 os를 설치 할 수 있음
ceilometer : 과금 서비스를 제공하는 구성 요소
heat : 다음 인스턴스 생성 때 자동으로 개발 인프라를 구축 할 수 있는 기능을 제공하는 요소
quantum : 오픈스택 서비스와 인터페이스 디바이스 간의 네트워킹 서비스를 지원하는 역할
